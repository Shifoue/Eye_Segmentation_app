Face segmentation

Context

This is a proof of concept about Eye Segmentation and the methods used.

The model was trained on the CelebFaces Kaggle's dataset and achieved 89,5% Dice Score on the test set.
In order to train the model, I created a train set of 800 images and a validation set of 200 images. I also created a test set of 1000 images to evaluate the model's performance.
Then I created the associated masks using Mediapipe. It allowed me to approximate eye position by applying mesh on the faces. Although the test set results are excellent, we could improve generalization by utilizing a dataset with more diversity.
I recommend using a dataset that includes various types of face pictures, rather than just close-up shots. This could significantly improve the generalization of the model on different face inputs.
The current model doesn't generalize well to all types of face pictures because of the CelebFaces dataset's limitations.

Explanations

This demonstration aims to showcase the results of the technical choices I made on the CelebFaces Kaggle's dataset and the generalization capabilities of the methods used.
To ensure a robust learning strategy for the model, I implemented a Dice Score loss. This approach effectively counters poor strategies, such as predicting only white pixels, by focusing solely on the segmented parts of the image..
Applying this loss function to higher quality datasets could enable the model to achieve even better results.
