Face segmentation

Context

This is a proof of concept about Eye Segmentation and the methods used.
The model was trained on the CelebFaces Kaggle's dataset and achieved 89,5% Dice Score on the test set.
In order to train the model, I created a train set of 800 images and a validation set of 200 images. I also created a test set of 1000 images to evaluate the model's performance.
Then I created the associated masks using Mediapipe. It allowed me to approximate eye position by applying mesh on the faces. Even though the results are great on the test set, we could achieve better generalization by using another more precisely annotated dataset.
I recommend using a dataset that includes various types of face pictures, rather than just close-up shots. This could significantly improve the generalization of the model on different face inputs.
The current model doesn't generalize well because of the CelebFaces dataset's limitation.

Explanations

This demonstration aims to showcase the results of the technical choices I made on the CelebFaces Kaggle Dataset and the generalization capabilities of the methods used.
For example the Dice Score loss is a great solution against bad learning strategies, enforcing a focus on the segmented part instead of falling into a white pixel only strategy.
This loss could be applied to other higher quality datasets allowing the model to achieve even better results on those datasets.
